# Babel插件

# Babel插件机制

插件通常是一些单一的功能模块，它们会在转换代码时对 AST（抽象语法树）进行操作，从而实现代码的转换。

Babel 可以很容易地通过插件机制进行扩展和自定义。

# AST解析流程

  • 词法分析：源码经过词法分析（分词）产生token序列，可看作一个扁平的语法片段数组
  • 语法分析：对token序列进行语法分析，这个阶段会使用token中的信息把它们转换成一个 AST 的表述结构
  • 转换：转换步骤接收 AST 并对其进行遍历，在此过程中对节点进行添加、更新及移除等操作。 这是 Babel 或是其他编译器中最复杂的过程 同时也是插件将要介入工作的部分。
  • 生成AST：深度优先遍历整个 AST，然后构建可以表示转换后代码的字符串

# 概念

# Visitors（访问者）

  • 访问节点：进入一个节点，其实就是在访问节点。
  • 访问者是一个用于 AST 遍历的跨语言的模式。 简单的说它们就是一个对象，定义了用于在一个树状结构中获取具体节点的方法。
  • 在遍历节点的过程中，每个节点都会有进入和退出两次访问机会
  • 示例: 把它用于遍历中时，每当在树中遇见一个 Identifier 的时候会调用 Identifier() 方法。
// Identifier() { ... } 是 Identifier: { enter() { ... } } 的简写形式。.
const MyVisitor = {
  Identifier() {
    console.log("Called!");
  }
};

const MyVisitor = {
  Identifier: {
    enter() {
      console.log("Entered!");
    },
    exit() {
      console.log("Exited!");
    }
  }
};

# Paths（路径）

  • Path: 表示两个节点之间连接的对象。
  • 路径对象包含添加、更新、移动和删除节点有关的其他很多方法
  • 路径是一个节点在树中的位置以及关于该节点各种信息的响应式 Reactive 表示
  • 示例：
{
  type: "FunctionDeclaration",
  id: {
    type: "Identifier",
    name: "square"
  },
  ...
}

# Paths in Visitors（存在于访问者中的路径）

  • 当你有一个 Identifier() 成员方法的访问者时，你实际上是在访问路径而非节点。
  • 通过这种方式，你操作的就是节点的响应式表示（译注：即路径）而非节点本身。

# 核心库

# @babel-core

包含 @babel/parser, @babel/traverse, @babel/generator，其中 parser 负责将源代码解析成 ast，traverse 向外提供遍历 ast 节点的能力，generator 负责根据变换后的 ast 生成目标代码。

# @babel/traverse

Babel Traverse（遍历）模块维护了整棵树的状态，并且负责替换、移除和添加节点。

# @babel/types

Babel Types模块是一个用于 AST 节点的 Lodash 式工具库，它包含了构造、验证以及变换 AST 节点的方法。

# @babel/generator

Babel Generator模块是 Babel 的代码生成器，它读取AST并将其转换为代码和源码映射（sourcemaps）

# @babel/template

babel-template 是另一个虽然很小但却非常有用的模块。 它能让你编写字符串形式且带有占位符的代码来代替手动编码， 尤其是生成的大规模AST的时候。

# 栗子2

  • 去掉代码中的cosole 首先查看语句console.log的AST解析结果
console.log('');
1

  • console的最近父节点是MemberExpression
  • 在父节点中查找console语句
  • console.log属于ExpressionStatement，找到这句话的节点
  • 删除这句话
module.exports = function ({ types: t }) {
  return {
    visitor: {
      // console.log的最近父节点是MemberExpression
      MemberExpression(path) { 
        // 查找console语句
        if (!path.get('object').isIdentifier({ name: 'console' })) return;
        // console.log属于ExpressionStatement，找到这句话的节点
        const parent = path.findParent((path) => path.isExpressionStatement());
        // 删除这句话
        parent.remove();
      }
    }
  };
};

# 栗子3

  • @babel/plugin-transform-arrow-functions (opens new window) 源码
  • 主要使用了path自带的isArrowFunctionExpression和arrowFunctionToExpression方法
import { declare } from "@babel/helper-plugin-utils";

export interface Options {
  spec?: boolean;
}

export default declare((api, options: Options) => {
  api.assertVersion(7);

  const noNewArrows = api.assumption("noNewArrows") ?? !options.spec;

  return {
    name: "transform-arrow-functions",

    visitor: {
      ArrowFunctionExpression(path) {
        // In some conversion cases, it may have already been converted to a function while this callback
        // was queued up.
        if (!path.isArrowFunctionExpression()) return;

        if (process.env.BABEL_8_BREAKING) {
          path.arrowFunctionToExpression({
            // While other utils may be fine inserting other arrows to make more transforms possible,
            // the arrow transform itself absolutely cannot insert new arrow functions.
            allowInsertArrow: false,
            noNewArrows,
          });
        } else {
          path.arrowFunctionToExpression({
            allowInsertArrow: false,
            noNewArrows,

            // This is only needed for backward compat with @babel/traverse <7.13.0
            // @ts-ignore(Babel 7 vs Babel 8) Removed in Babel 8
            specCompliant: !noNewArrows,
          });
        }
      },
    },
  };
});

# 使用插件

使用Babel插件的基本步骤：

  • 安装插件：使用npm或yarn安装需要的插件。
  • 配置插件：在项目根目录下创建babel.config.js或者.babelrc文件，并进行相关插件的配置。
  • 运行Babel：使用Babel将代码转换为目标版本。

例如，实现以下代码的转换：

const a = () => {
  console.log("Hello World!");
};

export default a;

# 插件顺序

如果两个转换插件都将处理“程序（Program）”的某个代码片段，则将根据转换插件或 preset 的排列顺序依次执行。

  • 插件在 Presets 前运行。
  • 插件顺序从前往后排列。
  • Preset 顺序是颠倒的（从后往前）。
babel.config.json
{
  "plugins": ["A", "B"]
}
