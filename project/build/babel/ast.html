# AST

# AST基础概念

AST（Abstract syntax tree）即抽象语法树，是计算机科学中的术语。它以树状结构表示编程语言的语法结构，书上的每个节点表示源代码中的一种结构。

一般由解析器将源代码进行分解，产生词法单元（token），再将词法单元解析成语法树AST，然后编译语法树AST到机器指令等待执行。

# 工具介绍

为了更好的了解使用AST，本文推荐使用acorn (opens new window) 作为AST解析器的学习工具。acorn是一个使用JavaScript实现的，小型JavaScript解析器。其基本语法如下：

let acorn = require('acorn')
let src = 1 + 2
console.log(acorn.parse(src))

acorn语法解析器的流程如下：

  1. 词法分析：源码经过词法分析（分词）产生token序列，可看作一个扁平的语法片段数组
[
  { type: { ... }, value: "n", start: 0, end: 1, loc: { ... } },
  { type: { ... }, value: "*", start: 2, end: 3, loc: { ... } },
  { type: { ... }, value: "n", start: 4, end: 5, loc: { ... } },
  ...
]

  1. 语法分析：对token序列进行语法分析，这个阶段会使用token中的信息把它们转换成一个 AST 的表述结构
  2. 转换：转换步骤接收 AST 并对其进行遍历，在此过程中对节点进行添加、更新及移除等操作。 这是 Babel 或是其他编译器中最复杂的过程 同时也是插件将要介入工作的部分。
  3. 生成AST：深度优先遍历整个 AST，然后构建可以表示转换后代码的字符串

Token的类型有：

  1. 关键字：import let return
  2. 变量名称：a、b
  3. 运算符号：+、-、*、/
  4. 结束符
  5. 状态机：对源码中的字符进行其含义的状态机判断。例如对于“/”符号的处理，对于源码4/2，“/”表示除法运算符号；而对于var re = /ab+c/，“/”表示正则表达式的起始字符。

我们可以将一份源码分为以下3个元素层次。一份源码即一个program，一个program由一个或多个statement组成，每个statement又由一个或多个expression组成。对于源码产生的AST也是由这三个层次的元素组成。

  • program：整个程序
  • statement: 语句
  • expression：表达式
