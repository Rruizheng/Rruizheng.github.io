# Babel预设

# 概念

Babel 预设（Presets）是一种预先定义好的一组插件集合，使用预设可以方便地进行对特定环境的转换。

比如，@babel/preset-env 是一个专门用来转换最新的 ES6/ES7/ES8 语法到更低级别的 JavaScript 语法的预设。

  • 与插件的区别
    • 插件是单个功能的模块，而预设是包含多个插件的集合。开发者可以根据需要自定义一个或多个预设集合，并把它们配置到 Babel 中实现项目需求。

# Babel官方基础预设

Babel 官方提供了一些基础预设包，可以帮助开发者自动执行插件和转换。

  1. @babel/preset-env：这个预设包可以根据目标环境的不同自动调整所需的插件和转换。它能够根据提供的配置选项确定转换的规则，用于转换 ECMAScript 新特性，解析模块语法，以及转换高级语言特性（如 async/await 和 generators）。同时，它也支持浏览器和被转译的 JavaScript 版本的配置。

  2. @babel/preset-react：这个预设包可以使我们不用通过手动配置就可以使用 JSX，同时，它还包含了一些用于处理 React 代码的插件。

  3. @babel/preset-typescript：这个预设包可以将 TypeScript 代码转换为 JavaScript，包括类型检查和其他的 TypeScript 特定功能。

  4. @babel/preset-flow：这个预设包可以将 Flow 代码转换为 JavaScript。

  5. @babel/preset-env 包含一个集合对象 @babel/plugin-transform-runtime，它会将生成的代码与运行时连接起来，以便在需要的情况下避免重复的工作。 这样可以减少源代码和运行时的大小。等等。

以上预设包都可以通过 npm 安装，然后在 Babel 配置文件中配置，以自动执行插件和转换。

这些预设包能够帮助开发者快速地完成常用代码的转换，同时也为配置 Babel 提供了方便。如果要转换特定的 ECMAScript 版本，或将 TypeScript 代码转换为 JavaScript 代码，最好使用相应的预设包。

# 创建一个预设

如需创建一个自己的预设（无论是为了本地使用还是发布到 npm），需要导出（export）一个配置对象。

module.exports = function() {
  return {
    plugins: ["pluginA", "pluginB", "pluginC"],
  };
};

preset 可以包含其他的 preset，以及带有参数的插件。

module.exports = () => ({
  presets: [require("@babel/preset-env")],
  plugins: [
    [require("@babel/plugin-proposal-class-properties"), { loose: true }],
    require("@babel/plugin-proposal-object-rest-spread"),
  ],
});

# 使用预设

  • 加载顺序：为了确保向后兼容，采用逆序排列（从后往前）
    • C -> B -> A
// babel.config.json
{
  "presets": ["A", "B", "C"]
}

// preset 参数配置
{
  "presets": [
    [
      "@babel/preset-env",
      {
        "loose": true,
        "modules": false
      }
    ]
  ]
}
