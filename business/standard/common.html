# 命名规则

在计算机科学中有两件难事：缓存失效和命名。

起⼀个好名字应该很难，因为⼀个好名字需要把要义浓缩在⼀到两个词中

# 命名规则

• 驼峰命令法(CamelCase)：例如：“myName”

函数名中的每⼀个逻辑断点都有⼀个⼤写字⺟来标记。

• 帕斯卡命名法（PascalCase）：例如：“MyName” ⼤驼峰命名法。 每个单词的第⼀个字⺟都⼤写。

• 匈⽛利命名法 标识符的名字以⼀个或者多个⼩写字⺟开头作为前缀； 前缀之后的是⾸字⺟⼤写的⼀个单词或多个单词组合，该单词要指明变量的⽤途

# 变量名

变量名应该是名词，能够正 确地描述业务，有表达⼒。 果⼀个变量名需要注释来补 充说明，那么很可能说明命 名就有问题。

# 函数名

函数命名要具体，空泛的命 名没有意义

# 类名

  • 实体类: 承载了核⼼业务数据和核⼼业务逻辑，其命名要充分体现业务语义，并 在团队内达成共识，如Customer、Bank 和Employee等。

  • 辅助类: 要能够通过后缀来体现功能。例 如，⽤来为Customer做控制路由的控制 类CustomerController、提供Customer 服务的服务类CustomerService、获取数 据存储的仓储类CustomerRepository。

# 包名

包名应该能够反映⼀组类在 更⾼抽象层次上的联系。包 的命名要适中，不能太抽 象，也不能太具体，不然会 失去包⽤来限定范围的作 ⽤。

# 模块名

⼀⽅⾯，其名称保证了模块 在仓库中的唯⼀性； 另⼀⽅⾯，名称要反映模块 在系统中的职责。

# 命名⼀致性

保持命名的⼀致性，可以提⾼代码的可读性，从⽽简化复杂度。 因此，我们要⼩⼼选择命名，⼀旦选中，就要持续遵循，保证名称始终⼀致。

# JS代码常见规范

  1. 缩进和空格：

    • 使用 2 或 4 个空格作为缩进。
    • 在运算符周围和逗号后面使用空格。
    • 在大括号前放置一个空格。

  2. 命名规范：

    • 使用驼峰命名法（camelCase）给变量、函数和方法命名。
    • 使用大写驼峰命名法（PascalCase）给类和构造函数命名。
    • 使用全大写字母和下划线（UPPER_CASE）命名常量。

  3. 注释：

    • 使用 // 或 /* ... */ 进行单行或多行注释。
    • 在代码中使用注释解释关键代码段的用途。

  4. 变量声明：

    • 使用 let 或 const 来声明变量，避免使用 var。
    • 使用 const 声明不会被重新赋值的变量。
    • 在变量声明时立即初始化变量。

  5. 函数：

    • 使用函数表达式或箭头函数来定义函数。
    • 在函数之间添加空行，以提高可读性。
    • 使用函数参数的默认值来简化代码。

  6. 字符串：

    • 使用单引号或双引号来定义字符串。
    • 在字符串拼接时使用模板字符串（使用反引号 `）。

  7. 数组和对象：

    • 对象和数组的键和值之间使用冒号分隔。
    • 在对象和数组的末尾不要添加多余的逗号。

  8. 条件语句和循环：

    • 使用严格相等运算符（===）而不是相等运算符（==）。
    • 在条件语句和循环中使用大括号，并在打开的括号前添加一个空格。

  9. 错误处理：

    • 使用 try-catch-finally 块来捕获和处理错误。

  10. 模块化：

    • 使用模块化的方式组织代码，使用 import 和 export 关键字导入和导出模块。

这些是一些通用的 JavaScript 代码规范，可以帮助提高代码的可读性和维护性。当然，具体的规范还可以根据团队或项目的需求进行调整。

# CSS常见规范

  1. 缩进和空格：

    • 使用 2 或 4 个空格作为缩进。
    • 在选择器和属性之间使用空格。
    • 在属性值的冒号后面添加一个空格。

  2. 命名规范：

    • 选择器、属性和值都使用小写或驼峰方式
    • 使用有意义的类名和 ID 名称，避免使用无意义的缩写。
    • 使用连字符（-）作为多个单词之间的分隔符。

  3. 属性声明顺序：

    • 按照特定的顺序声明 CSS 属性，显示定位布局 -> 盒模型等自身属性 -> 变形类属性 -> 文本类及修饰类 -> 行为辅助类，例如：布局属性、盒模型属性、文本属性等。
    • 将相关的属性分组，并使用空行分隔。

  4. 不论是单独的样式表，还是合并后的样式表，内部结构都应该遵行基础样式->组件样式 -> 私有样式 -> 皮肤样式 的组合顺序。

  5. 媒体查询：

    • 将媒体查询放在样式表的末尾，以避免覆盖其他样式。
    • 使用媒体查询时，将相关规则包装在适当的媒体查询块中。

  6. 项目内部约定特定的命名空间，例如：功能（function）(.f-)，布局（grid）(.g-)，动画（animition）(.a-)

  7. 其他常用约定

  • 最后一个值也以分号结尾，如{ margin:0; padding:0; }；
  • 省略值为 0 时的单位，如：0px、0em、0% 等值都缩写为 0；
  • 使用单引号，或省略引号，如：content:'.';， background:url(bg.png);
  • 注释格式：单行注释，多行注释，注释分隔线
