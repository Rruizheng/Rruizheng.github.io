# ESLint & Prettier

ESLint 和 Prettier 都是用于提高代码质量和一致性的工具，但它们的功能和应用方式略有不同。

  1. ESLint：

    • 作用：ESLint 是一个静态代码分析工具，用于检查 JavaScript 代码中的潜在问题和错误，以及规范化代码风格。
    • 功能：
      • 检查代码中的语法错误和潜在的 bug。
      • 强制执行一致的代码风格和规范，如变量命名、缩进、空格等。
      • 支持通过插件和配置文件自定义规则和扩展功能。

  2. Prettier：

    • 作用：Prettier 是一个代码格式化工具，用于自动化格式化和美化代码，使其具有一致的风格。
    • 功能：
      • 格式化代码的缩进、空格、换行等风格，使代码具有统一的外观。
      • 支持多种编程语言，包括 JavaScript、CSS、HTML 等。
      • 具有可配置的选项，可以根据项目需求进行自定义。

区别：

  • ESLint 是一个静态代码分析工具，主要用于检查代码质量和规范性，可以发现和修复潜在的问题和错误。它可以通过配置文件自定义规则和检查的范围。
  • Prettier 是一个代码格式化工具，主要用于统一代码的风格和外观。它强制执行一套默认的格式化规则，不需要手动配置即可自动格式化代码。

通常情况下，ESLint 和 Prettier 可以结合使用，ESLint 负责检查代码质量和规范性，Prettier 负责统一代码的风格。可以通过配置文件和插件将两者集成在项目中，以实现自动化的代码检查和格式化。这样可以提高代码质量、一致性和可读性，并减轻开发者在维护代码风格上的负担。

# 2、配置文件.eslintrc.js

# 普通版：

// .eslintrc.js
module.exports = {
    "env": {
        "browser": true,
        "commonjs": true,
        "es6": true,
        "jquery": true
    },
    extends: ['eslint:recommended'],
    parserOptions: {
        ecmaVersion: 6, // 指定要使用的ECMAScript6版本
        sourceType: 'module'
    },

    plugins: ['prettier'],

    rules: {
        'arrow-parens': ['error', 'as-needed'], // 箭头函数参数只有一个时，可以省略参数的括号，否则error提示

        'no-debugger': process.env.NODE_ENV === 'production' ? 'error' : 'off', // 生产环境中不允许使用debugger

        'no-console': 'off', // 允许使用console

        'prettier/prettier': 'warn', // 对于prettier报错进行warn提醒
    }
}

# React版：

module.exports = {
    extends: ['react-app', 'eslint:recommended'],
    parser: "babel-eslint",
    parserOptions: {
        ecmaVersion: 6, // 指定要使用的ECMAScript6版本
        sourceType: 'module',
        ecmaFeatures: { 'jsx': true } //启用JSX
    },

    plugins: ['prettier', 'react-hooks'],

    rules: {
        'arrow-parens': ['error', 'as-needed'], // 箭头函数参数只有一个时，可以省略参数的括号，否则error提示

        'no-debugger': process.env.NODE_ENV === 'production' ? 'error' : 'off', // 生产环境中不允许使用debugger

        'no-console': 'off', // 允许使用console

        'react/jsx-no-target-blank': 'off', // a链接允许直接使用_blank

        'prettier/prettier': 'warn', // 对于prettier报错进行warn提醒

        'jsx-a11y/anchor-has-content': 'off',

        "react-hooks/rules-of-hooks": "error",

        "react-hooks/exhaustive-deps": "warn",

        'jsx-a11y/anchor-is-valid': 'off'
    }
}

# 3、配置.prettierrc.js：

// .prettierrc.js
module.exports = {
    printWidth: 150, // 设置prettier单行输出（不折行）的（最大）长度

    tabWidth: 4, // 设置工具每一个水平缩进的空格数

    useTabs: false, // 使用tab（制表位）缩进而非空格

    semi: true, // 在语句末尾添加分号

    singleQuote: true, // 使用单引号而非双引号

    trailingComma: 'none', // 在任何可能的多行中输入尾逗号

    bracketSpacing: true, // 在对象字面量声明所使用的的花括号后（{）和前（}）输出空格

    arrowParens: 'avoid', // 为单行箭头函数的参数添加圆括号，参数个数为1时可以省略圆括号

    jsxBracketSameLine: true, // 在多行JSX元素最后一行的末尾添加 > 而使 > 单独一行（不适用于自闭和元素）

    rangeStart: 0, // 只格式化某个文件的一部分

    rangeEnd: Infinity, // 只格式化某个文件的一部分

    filepath: 'none', // 指定文件的输入路径，这将被用于解析器参照

    requirePragma: false, // (v1.7.0+) Prettier可以严格按照按照文件顶部的一些特殊的注释格式化代码，这些注释称为“require pragma”(必须杂注)

    insertPragma: false, //  (v1.8.0+) Prettier可以在文件的顶部插入一个 @format的特殊注释，以表明改文件已经被Prettier格式化过了。

    proseWrap: 'preserve', // (v1.8.2+)

    jsxBracketSameLine: false, // jsx闭合标签在同一行
}
