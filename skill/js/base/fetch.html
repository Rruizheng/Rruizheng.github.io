# fetch

Fetch API 和 XMLHttpRequest 是两种不同的 JavaScript API，用于进行网络请求。它们有以下几个主要的区别：

  1. 语法和用法：Fetch API 使用起来更简洁和直观。它使用 fetch() 函数发送请求，并返回一个 Promise 对象。而 XMLHttpRequest 使用较为繁琐的 API，需要创建一个 XMLHttpRequest 对象，然后设置各种属性和回调函数来发送和处理请求。

  2. 异步处理：Fetch API 默认使用 Promise 和 async/await 来处理异步操作，使代码更具可读性和易于维护。而 XMLHttpRequest 使用回调函数来处理请求的异步操作，需要编写多个回调函数，使得代码结构较为复杂。

  3. 请求和响应对象：Fetch API 使用 Request 和 Response 对象来表示请求和响应。这些对象提供了更多的方法和属性，使得处理请求和响应更加灵活和方便。而 XMLHttpRequest 使用自身的对象来表示请求和响应，方法和属性较为有限。

  4. 跨域请求：Fetch API 默认不发送 Cookie 和认证信息，需要显式设置 credentials 属性为 'include' 才能发送和接收 Cookie。而 XMLHttpRequest 默认发送和接收 Cookie，可以通过设置 withCredentials 属性来控制是否发送和接收 Cookie。

  5. 兼容性：Fetch API 在较新的浏览器中得到了广泛支持，但在一些旧版本的浏览器中可能不被支持。而 XMLHttpRequest 在大多数主流浏览器中都得到了支持，包括较旧的浏览器。

总的来说，Fetch API 是一种更现代和推荐的网络请求方式，它提供了更简洁、灵活和强大的功能。但如果需要在旧版本的浏览器中使用，或需要更精细的控制和处理网络请求，可以选择使用 XMLHttpRequest。

# 缓存模式

在 Fetch API 中，可以通过配置 cache 属性来设置缓存模式。cache 属性用于指定请求的缓存行为。

以下是几种常见的缓存模式：

  1. 默认模式（default）：根据浏览器的默认行为来决定是否缓存请求和响应。如果满足缓存条件，则使用缓存。这是默认的缓存模式。
fetch(url, {
  cache: 'default'
})
  .then(response => {
    // 处理响应
  })
  .catch(error => {
    // 处理错误
  });
