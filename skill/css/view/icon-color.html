    # 图标着色

    总结了一下在开发过程中常用的图标换色方案。

    # 1. 使用mask-image

    使用mask-image遮罩来进行图标着色，是当前比较常用的方案。

    • 原理：所谓遮罩，就是原始图片只显示遮罩图片非透明的部分。将需要换肤的icon作为遮罩，元素本身设置背景色作为原始图片，就可以实现icon换色效果。
    • 注意：要求换肤使用的icon为非透明icon。若图片本身带有透明度，则投射图像也会带有透明度。
    
import React from 'react';
import PropTypes from 'prop-types';
import './index.less';

const MIcon = (props) => {
    const {
        url, bgColor, width, height, className
    } = props;

    const divstyle = {
        WebkitMaskImage: `url(${url})`,
        maskImage: `url(${url})`,
        backgroundColor: bgColor,
        width,
        height,
    };

    return (
        <span
            className={`m-micon ${className}`}
            style={divstyle} />
    );
};

MIcon.propTypes = {
    className: PropTypes.string,
    url: PropTypes.string.isRequired,
    bgColor: PropTypes.string.isRequired,
    width: PropTypes.number.isRequired,
    height: PropTypes.number.isRequired,
};

MIcon.defaultProps = {
    className: ''
};

export default MIcon;

// index.less
.m-micon {
    display: inline-block;
    -webkit-mask-size: 100% 100%;
    mask-size: 100% 100%;
}

    # 2.使用drop-shadow

    使用filter属性的drop-shadow，投射阴影。

    • 原理：drop-shadow滤镜可以给元素或图片非透明区域添加投影，通过设置投影颜色，即可实现图标变色效果。

      • 在Chrome浏览器下，如果一个元素的主体部分，无论以何种方式，只要在页面中不可见，其drop-shadow是不可见的；实体部分哪怕有1像素可见，则drop-shadow完全可见。
      • 如果实体部分也在可视区域内，但是是透明的，图片也是可见的。所以可以使用borderRight: ${width} solid transparent，使用透明边框使元素始终有部分在视窗内部，保持元素可见。

    • 问题：在chrome、safari等浏览器中，图标显示是没有问题的。但在ios的webview内部，会出现实体部分隐藏的时候，投射部分也无法显示的情况。

      • 解决方法1：对ios，需要增加transform: translateZ(0)；添加到gpu复合层渲染就正常了，但安卓，增加了这个属性会有问题。所以这里要判断一下。
      • 解决方法2：不要使用overflow:hidden 可以试着想办法用其他元素遮盖主元素。

    • 注意：要求换肤使用的icon为非透明icon。若图片本身带有透明度，则投射图像也会带有透明度。显示效果与mask-image相同。

    感觉这个属性用来做图标换肤的时候兼容性还是有点问题，首选mask-image。

    import React from 'react';
import PropTypes from 'prop-types';
import './index.less';

const CIcon = (props) => {
    const {
        url, bgColor, width, height, className, isDark
    } = props;

		const iconDelStyle = {
        backgroundImage: `url(${url})`,
        filter: `drop-shadow(${width} 0 0 ${bgColor})`,
        WebKitFilter: `drop-shadow(${width} 0 0 ${bgColor})`,
        left: `-${width}`,
        borderRight: `${width} solid transparent`,
    };

		if (isIos()) {
				iconDelStyle.**transform = 'translateZ(0)';**
		}
    const iconStyle = {
        width,
        height
    };

    const main = (
        <i className="i-icon" style={iconStyle}>
            <i
                className={`i-del ${className}`}
                style={iconDelStyle} />
        </i>
    );

    return main;
};

CIcon.propTypes = {
    className: PropTypes.string,
    url: PropTypes.string.isRequired,
    bgColor: PropTypes.string.isRequired,
    width: PropTypes.number.isRequired,
    height: PropTypes.number.isRequired,
};

CIcon.defaultProps = {
    className: ''
};

export default CIcon;

// index.less
.i-icon {
    display: inline-block;
    position: relative;
		overflow: hidden;

    .i-del {
        height: 100%;
        width: 100%;
        position: relative;
        display: inline-block;
        background-size: cover;
        background-repeat: no-repeat;
    }
}

    # 3. svg元素换色

    SVG图片也是可以用上述两种方案实现换色，另外SVG本身的换色，推荐阅读下面文章

    • SVG图标颜色文字般继承与填充 (opens new window)，SVG内联图标颜色CSS可控制，SVG图标元素上是没有fill属性的，可以通过设置fill实现着色
    • SVG Sprites技术介绍 (opens new window)，原理是将SVG图标图形整合在一起，实际呈现的时候准确显示特定图标，在HTML层面，图标使用的代码成本，跟传统的CSS Sprite或者流行的font-face几乎无异，代码简洁，而且很好维护。所有的SVG图标都在一个SVG源上。
